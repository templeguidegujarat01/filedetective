About

Built to be inspected, not just used

FileDetective exists because inspecting a file shouldn't require handing it over to a stranger's server. It's a small, focused tool: point it at a file, and it tells you exactly what the browser can honestly determine about it — no more, no less.

Mission

Give anyone — students, journalists, IT professionals, or someone just curious about a download — a fast, trustworthy way to look inside a file without installing software or trusting a third-party upload service.

Privacy philosophy

Every feature is designed around a single constraint: the file cannot leave the device it's opened on. That constraint shapes the entire architecture, from the lack of a backend to the way reports are exported through the browser's own print dialog rather than a server-rendered PDF.

Technology

The project runs entirely as static files — HTML, CSS, and vanilla JavaScript — hosted on GitHub Pages. There's no account system, no database, and no hidden infrastructure. What you see in the browser is the whole application.

Transparency

Where a browser genuinely cannot expose certain information — like camera EXIF data or true malware signatures — FileDetective says so plainly instead of guessing or fabricating a result.

Roadmap

Development is organized in stages, each adding capability without compromising the privacy model.

Version 1

Production release

Core analysis for images, PDFs, audio, video, and text, with a polished, responsive interface.

Version 2

Advanced browser analysis

Batch analysis, additional hash algorithms, local history, and richer metadata extraction.

Version 3

Professional tool

Magic-byte verification, entropy estimation, a hex viewer, and file comparison.
