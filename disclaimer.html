What FileDetective can and can't tell you

Last updated July 2026

FileDetective is a metadata and hashing tool, not a security product. It does not detect malware, viruses, or malicious code of any kind.

Browser limitations

All analysis is performed using standard Web APIs available to JavaScript running in your browser. Browsers do not expose the same depth of information that dedicated desktop forensic software can access — for example, full EXIF camera data, deep binary structure, or embedded macros are outside what a browser can reliably read.

No fabricated data

When the browser cannot determine a value — such as a video's exact frame rate or a PDF's embedded fonts — FileDetective reports it as "Not available" rather than guessing or inventing a plausible-looking number.

Security checks are heuristic only

Flags like "double extension" or "executable file type" are pattern checks on the filename, not an analysis of the file's actual behavior. A file with no warnings is not guaranteed to be safe, and a file with a warning is not guaranteed to be harmful.

Estimated values

Some figures, such as audio bitrate, are calculated estimates derived from file size and duration rather than values read directly from the file's encoding — these are labeled "approximate" wherever shown.

Use your judgment

Treat FileDetective's output as one input among several when evaluating a file. For files from an untrusted source, use dedicated antivirus or security software in addition to any information shown here.
