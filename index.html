What it reads

One tool, every common file type

Images

Dimensions, aspect ratio, transparency, estimated megapixels, and file signature for PNG, JPG, GIF, WEBP, BMP, SVG, and more.

PDFs

Page count, author, producer, PDF version, encryption status, and document metadata — read directly from the file structure.

Audio

Duration, estimated bitrate, sample rate, and channel information for MP3, WAV, OGG, and AAC files.

Video

Duration, resolution, aspect ratio, and estimated frame rate for MP4, WEBM, and MOV files.

Code & data

Validation and structure for JSON, XML, and HTML, plus line, word, and character counts for plain text.

Security signals

Double-extension checks, executable detection, and filename heuristics — clearly labeled, never overstated.

How it works

Three steps, zero uploads

01

Choose a file

Click the scan frame or drag a file onto it. Nothing is sent anywhere — the browser reads the file directly from your device's memory.

02

Local analysis runs

JavaScript running on this page parses the file, computes a SHA-256 hash, and extracts whatever metadata the browser exposes.

03

Review and export

Results appear as a structured report you can copy, print, or export — and the file itself is discarded the moment you leave the page.

Privacy

Your file never leaves your device

FileDetective has no backend. There is no server to upload to, so there's nothing to intercept, store, or leak.

No uploads

Every byte of your file is read and processed by JavaScript running locally in this browser tab.

No tracking of files

Filenames, contents, and analysis results are never sent to any analytics or logging service.

No cloud storage

Nothing is saved outside your own device unless you explicitly export a report.

No hidden scanning

FileDetective only performs the checks described on this page — nothing runs silently in the background.

About

Built to be inspected, not just used

FileDetective exists because inspecting a file shouldn't require handing it over to a stranger's server. It's a small, focused tool: point it at a file, and it tells you exactly what the browser can honestly determine about it — no more, no less.

The project runs entirely as static files on GitHub Pages. There's no account system, no database, and no hidden infrastructure — what you see in the browser is the whole application.
