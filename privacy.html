Privacy Policy

Last updated: 20 July 2026

Summary

FileDetective analyzes files entirely inside your browser. There is no server that receives your files, no database that stores them, and no third party that scans them. This page explains exactly what that means in practice.

What never leaves your device

  • The contents of any file you select or drop onto FileDetective.
  • Filenames, extracted metadata, and computed hashes.
  • Any report you generate, unless you choose to export or share it yourself.

How analysis actually works

When you choose a file, your browser reads it into memory using standard Web APIs such as FileReader, Web Crypto, and the HTML media elements. All parsing, hashing, and metadata extraction happens in JavaScript running on this page. The file is discarded from memory when you close or refresh the tab.

Hosting and static assets

This site is hosted as static files on GitHub Pages. Loading the page itself requests HTML, CSS, JavaScript, and font files from GitHub's infrastructure and Google Fonts, the same way any website does. These requests do not include your file contents.

Analytics and tracking

FileDetective does not run analytics scripts, advertising trackers, or third-party pixels. No behavioral data about how you use the tool is collected.

Local storage

The current version of FileDetective does not persist analysis history between sessions. Any future local-only history feature would store data solely in your browser, never on a server, and would be clearly disclosed.

Changes to this policy

If this policy changes, the update will be reflected on this page along with a new "last updated" date.

Contact

Questions about this policy can be sent to hello@filedetective.app or raised on GitHub.
